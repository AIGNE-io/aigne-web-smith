# 您的第一个网站

本教程提供了一个直接的、实践性的指南，用于生成一个完整的多页面网站。该过程包括通过回答几个问题创建一个简单的配置文件，然后运行一个单一命令。无需任何技术经验。

在本指南结束时，您将拥有一套本地保存的网页，由 AI 根据您的具体要求生成，可供审查和发布。

## 准备工作

在继续之前，请确保您已成功安装 AIGNE CLI。如果您尚未安装 CLI，请按照[安装](./getting-started-installation.md)指南中的说明进行操作。

## 步骤 1：创建项目目录

首先，在您的计算机上创建一个新文件夹，用于存储您网站的配置和生成的文件。打开您的终端或命令提示符并运行以下命令：

```bash
# 为您的项目创建一个新目录
mkdir my-first-website

# 进入新目录
cd my-first-website
```

所有后续命令都应在此 `my-first-website` 目录中运行。

## 步骤 2：运行生成命令

整个网站生成过程由一个单一命令启动。在您的终端中，运行以下命令：

```bash aigne web generate icon=material-symbols:rocket-launch-outline
aigne web generate
```

因为这是一个新项目，WebSmith 将检测到不存在配置文件，并会自动启动一个交互式设置向导来帮助您创建一个。

## 步骤 3：完成配置向导

设置向导将引导您回答一系列问题，以了解您想要构建的网站类型。以下是您需要提供的信息的细分。

1.  **网站目的**：选择您网站的主要目标（例如，产品展示、文档中心、营销网站）。这有助于 AI 适当地定制结构和内容。
2.  **目标受众**：指明谁将访问您的网站（例如，客户、开发人员、企业主）。这会影响生成文本的语气和复杂性。
3.  **网站规模**：选择您需要的大致页面数量（例如，最简、标准、全面）。这决定了 AI 将创建的网站结构的深度。
4.  **主要语言**：选择您网站内容的主要语言。
5.  **翻译语言**：可选择您希望网站被翻译成的其他语言。
6.  **页面目录**：指定最终网站文件将保存到的文件夹。默认的 `aigne/web-smith/pages` 适用于大多数项目。
7.  **数据源路径**：提供 AI 应用作源材料的本地文件或目录的路径（例如，`./docs`、`../project-brief.md`）。如果您不提供任何路径，WebSmith 将分析您当前的目录并建议相关文件。
8.  **自定义规则**：添加您对网站的任何具体说明或要求，例如品牌指南或内容限制。

回答完所有问题后，WebSmith 会将您的选择保存到项目目录中一个名为 `config.yaml` 的文件中。

### 示例配置文件

您生成的 `config.yaml` 文件将与此类似。您可以随时直接编辑此文件以调整您网站的设置。

```yaml config.yaml icon=mdi:file-document-outline
# 用于页面发布的项目信息
projectName: My Awesome Project
projectDesc: This is a project that does amazing things.
projectLogo: ""
projectId: d2a8b9e7-f6c5-4d32-9a1b-0e8c7d6f5a4b
projectSlug: my-awesome-project

# =============================================================================
# 网站配置
# =============================================================================

# 目的：您希望读者实现的主要成果是什么？
# 可用选项（根据需要取消注释并修改）：
#   landingPage      - 落地页 / 主页：将访问者转化为用户或客户
#   ecommerce        - 电子商务 / 在线商店：在线销售产品或服务
#   portfolio        - 作品集 / 展示：展示创意作品、项目或成就
#   corporate        - 企业 / 商业：提供公司信息的专业商业网站
#   blog             - 博客 / 内容网站：分享文章、新闻和定期内容更新
#   saas             - SaaS / 软件产品：推广软件服务并引导用户上手
#   nonprofit        - 非营利 / 社区：推广事业、接受捐赠、吸引志愿者
#   education        - 教育 / 学习：提供课程、教程或教育内容
#   mixedPurpose     - 多用途网站：涵盖多种需求的综合性网站
pagePurpose:
  - saas

# 目标受众：谁会最常阅读这些内容？
# 可用选项（根据需要取消注释并修改）：
#   customers        - 客户 / 最终用户：购买或使用您产品/服务的人
#   businessOwners   - 企业主 / 创业者：寻找解决方案的经营者
#   marketers        - 营销团队：推广产品或管理活动的人员
#   designers        - 设计师 / 创意专业人士：专注于视觉设计和用户体验的人员
#   developers       - 开发人员 / 技术用户：构建或集成技术解决方案的人员
#   investors        - 投资者 / 利益相关者：评估商业潜力和增长的人员
#   jobSeekers       - 求职者 / 潜在雇员：寻找职业机会的人员
#   students         - 学生 / 学习者：寻求教育内容或资源的人员
#   generalPublic    - 公众 / 混合受众：兴趣和背景各异的广泛受众
targetAudienceTypes:
  - developers

# 网站规模：您的网站应包含多少页面？
# 可用选项（根据需要取消注释并修改）：
#   singlePage         - 单页（仅 1 页）：所有内容合并到 1 个页面
#   minimal            - 最简（2-6 页）：仅包含核心页面 - 快速上线
#   standard           - 标准（7-12 页）：包含主要部分的完整网站 [推荐]
#   comprehensive      - 全面（12+ 页）：功能齐全、包含详细部分的网站
#   aiDecide           - 让 AI 决定：分析项目复杂性并建议合适的规模
websiteScale: standard

# 自定义规则：定义特定的页面生成规则和要求
rules: ""

# 语言设置
locale: en

# 目录和源路径配置
pagesDir: aigne/web-smith/pages  # 保存生成页面的目录

# 数据源路径（关键！）- AI 用于生成内容的源材料
# 这是决定网站质量最重要的配置
sourcesPath:
  - ./README.md
  - ./docs
  - ./CHANGELOG.md

# 默认数据源 - 每个页面都包含的通用资源
defaultDatasources:
  - ./media.md
```

**重要提示：`sourcesPath` 是配置文件中最关键的部分。** 它指定了 AI 用于生成网站内容的所有源材料。没有足够、高质量的数据源，生成的网站内容将会是通用且不准确的。请确保包含所有相关的文档、描述和信息文件。

## 步骤 4：审查并批准网站计划

配置保存后，AI 将分析您的要求和数据源，为您的网站创建一个建议的结构。此计划将显示在您的终端中以供审查。

系统会提示您批准该计划或请求更改。对于这第一个网站，您可以直接批准建议的结构以继续。

## 步骤 5：查看您生成的网站

一旦您批准了计划，AI 将编写内容并为每个页面生成必要的模板文件。该过程可能需要几分钟，具体取决于页面数量。

生成的文件保存在您指定的目录中（例如，`aigne/web-smith/pages`），它们是用于发布的模板。要查看最终渲染的、将在线显示的网站，您必须首先发布它。此步骤将模板转换为一个实时网站。

## 总结

恭喜，您已成功使用 AIGNE WebSmith 生成了您的第一个网站。核心过程包括运行一个单一命令，回答一系列问题以创建配置文件，然后让 AI 根据您的输入构建网站。
