# 互動模式（Beta）

> **請注意：** 互動模式功能目前處於 Beta 階段。雖然功能齊全，但您可能會遇到非預期的行為。感謝您的回饋，我們將持續改善此體驗。

使用本指南啟動 `aigne web chat`，以自然語言與 WebSmith 對話，並讓助理代表您觸發生成／更新／發布操作。

`chat` 命令提供了一個互動式、對話式的介面，用於建立和管理您的網站。您無需執行 `generate` 或 `update` 等個別命令，只需用簡單的語言描述您的需求，AI 助理便會引導您完成整個過程。對於偏好更具引導性體驗的使用者來說，這是一個理想的起點。

## 運作方式

互動模式會話由一個專門的 AI agent 提供支援，該 agent 能夠理解您的請求並使用其他 WebSmith 工具來執行任務。它會記住您的對話內容，讓您可以進行連續的變更並在先前的操作基礎上繼續建構。

一般的工作流程遵循一個簡單的對話循環：

```d2
direction: down

start: {
  label: "開始互動模式會話"
  shape: oval
}

user_input: {
  label: "使用者提供請求\n（例如：『建立一個網站』）"
}

ai_process: {
  label: "AI 助理分析請求"
}

execute_task: {
  label: "AI 執行相應的命令\n（生成、更新、發布等）"
  shape: diamond
}

show_result: {
  label: "助理顯示結果\n並等待下一條指令"
}

end: {
  label: "結束互動模式會話"
  shape: oval
}

start -> user_input
user_input -> ai_process
ai_process -> execute_task
execute_task -> show_result: "成功"
show_result -> user_input
execute_task -> user_input: "需要更多資訊"
user_input -> end: "使用者輸入 'exit'"
```

## 開始互動式會話

若要開始，請在您的終端機中執行 `chat` 命令：

```bash 命令 icon=lucide:terminal
aigne web
```

這將啟動助理，它會向您打招呼並等待您的指令。

## 您可以做什麼

互動模式助理可以執行 AIGNE WebSmith CLI 中大部分的核心任務。您只需簡單地要求它：

*   **生成一個新網站**：描述您的網站需求、目標受眾和期望的風格。
*   **更新現有內容**：指定您想變更的頁面或區塊，並提供新的詳細資訊。
*   **翻譯頁面**：請求將特定頁面翻譯成另一種語言。
*   **發布您的網站**：告訴助理您準備好將網站發布到 Pages Kit。
*   **管理檔案**：要求查看目前的檔案結構或特定頁面的內容。

## 範例對話

以下是一個範例，說明如何使用互動模式來建立、修改和發布一個簡單的網站。

```bash 命令列 icon=lucide:terminal
aigne web

> 歡迎使用 AIGNE WebSmith 互動式助理！
> 今天我能如何協助您建立網站？

您：
我需要為一個新的 SaaS 產品建立一個網站，該產品幫助團隊管理他們的專案。

> 當然。可以請您提供更多一些細節嗎？
> - 產品的核心功能是什麼？
> - 目標受眾是誰？
> - 網站應該採用什麼風格（例如：商務、現代、活潑）？

您：
主要功能是任務追蹤、團隊協作和報告。目標受眾是中小型企業主。我們採用現代商務風格吧。

> 謝謝您。我已有足夠的資訊來規劃網站結構。
> 我現在將生成以下頁面：
> - 首頁
> - 功能
> - 定價
> - 關於我們
> - 聯絡我們
> ...
> [生成過程開始]
> ...
> 網站生成已完成。檔案已儲存至 'output' 目錄。
> 接下來您想做什麼？

您：
在首頁上，新增一個客戶見證的區塊。

> 了解。我將在首頁新增一個客戶見證區塊。
> [更新過程開始]
> ...
> 首頁已更新。還有其他需要嗎？

您：
沒有了，目前就這樣。請發布網站。

> 我現在將把網站發布到您設定的 Pages Kit 專案。
> [發布過程開始]
> ...
> 您的網站已成功發布。
> 您現在可以輸入 'exit' 來離開互動模式。

您：
exit

> 再見！
```

## 離開互動模式

若要隨時結束您的互動式會話，只需輸入 `exit` 或 `quit` 並按下 Enter 鍵。

---

互動模式是一種強大的方式，讓您無需記住特定的命令和參數即可管理整個網站生命週期。若想進行更直接的控制，您隨時可以使用後續章節中詳述的標準命令。
