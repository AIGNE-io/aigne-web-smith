# 環境設定の管理

このガイドでは、`aigne web prefs` の設定方法について説明します。保存されたルールのリスト表示、有効/無効の切り替え、不要になったエントリの削除方法などが含まれます。

AIGNE WebSmith は、あなたのフィードバックから学習し、ウェブサイトの生成・更新方法を洗練させます。これらの学習された指示は、ユーザーの環境設定として保存されます。`prefs` コマンドは、これらの保存された環境設定を表示、管理、消去する方法を提供し、AI の振る舞いを完全にコントロールできるようにします。

環境設定を管理することで、AI があなたの特定のスタイルガイド、コンテンツ要件、構造上の慣習に長期にわたって一貫して従うようになります。

## 環境設定のリスト表示

現在保存されているすべての環境設定を表示するには、`list` コマンドを使用します。このコマンドは、すべての環境設定ルールを、その ID、アクティブ状態、スコープ、およびルール自体を含むテーブルで表示します。

```bash 環境設定のリスト表示 icon=lucide:terminal
aigne web prefs list
```

エイリアス `ls` も使用できます：

```bash 環境設定のリスト表示 icon=lucide:terminal
aigne web prefs ls
```

### 出力例

出力は整形されたテーブル形式となり、保存されたルールを簡単に確認できます。

| ID | Active | Scope | Rule |
| :--------- | :----- | :-------- | :----------------------------------------------------------------- |
| pref_a1b2c3d4 | true | page | フォーマルでプロフェッショナルなトーンで記述する。 |
| pref_e5f6g7h8 | true | structure | 古い「レガシー API リファレンス」のページやセクションは生成しない。 |
| pref_i9j0k1l2 | false | theme | ヘルスケア関連のウェブサイトでは、暖色系のカラートーンを使用する。 |

## 環境設定の切り替え

環境設定を完全に削除せずに一時的に無効化したい場合は、`toggle` コマンドを使用できます。このコマンドは、ルールの `active` ステータスを切り替えます。特定のルールを切り替えるには、`--id` パラメータでその一意の ID を指定する必要があります。

```bash 環境設定の切り替え icon=lucide:terminal
aigne web prefs toggle --id <PREFERENCE_ID>
```

エイリアス `t` も使用できます：

```bash 環境設定の切り替え icon=lucide:terminal
aigne web prefs t --id <PREFERENCE_ID>
```

### 例

上記の例からテーマの環境設定を無効化するには：

```bash Prefs Toggle --id Pref_i9j0k1l2 icon=lucide:terminal
aigne web prefs toggle --id pref_i9j0k1l2
```

再度 `aigne web prefs list` を実行すると、`pref_i9j0k1l2` の `active` は `false` と表示されます。同じ ID で再度 toggle コマンドを実行すると、再び有効化されます。

## 環境設定の削除

不要になった環境設定を完全に削除するには、`remove` コマンドを使用します。この操作は元に戻せません。削除するルールを `--id` パラメータで ID を指定して特定する必要があります。

```bash 環境設定の削除 icon=lucide:terminal
aigne web prefs remove --id <PREFERENCE_ID>
```

エイリアス `rm` も使用できます：

```bash 環境設定の削除 icon=lucide:terminal
aigne web prefs rm --id <PREFERENCE_ID>
```

### 例

上記の例からページの環境設定を完全に削除するには：

```bash Prefs Remove --id Pref_a1b2c3d4 icon=lucide:terminal
aigne web prefs remove --id pref_a1b2c3d4
```

指定された環境設定は、`preferences.yml` ファイルから削除されます。

## 環境設定スコープの理解

環境設定は、割り当てられた `scope` に基づいて適用されます。スコープは、ルールがどのコンテキストでトリガーされるかを決定します。

| Scope | Description |
| :------------ | :------------------------------------------------------------------------------------------------------ |
| **global** | 生成とコンテンツ洗練のすべての段階に適用されます。普遍的なスタイルやコンテンツのルールに使用します。 |
| **structure** | ウェブサイトの構造を計画または更新する場合（例：ページの追加、削除）にのみ適用されます。 |
| **page** | 個々のページのコンテンツを生成または洗練する場合に適用されます。 |
| **translation** | コンテンツの翻訳プロセス中にのみ適用されます。 |
| **theme** | ウェブサイトのビジュアルテーマ（色やフォントなど）を生成または変更する場合に適用されます。 |

## まとめ

`prefs` コマンドは、AIGNE WebSmith の長期的な振る舞いをカスタマイズし、制御するための不可欠なツールです。環境設定をリスト表示、切り替え、削除することで、クリーンで効果的なルールセットを維持し、AI がプロジェクトの特定のニーズに合った結果を一貫して生成することを保証できます。
