# 更新页面

当您需要对单个页面运行 `aigne web update` 时——重写部分内容、插入新模块、重新排序区块或删减过时内容——而不想更改网站的其余部分，请使用本指南。

此过程与修改整个站点地图不同。有关添加、删除或重组页面的说明，请参阅[更新结构](./guides-update-website-update-structure.md)指南。

## 更新过程

内容更新通过 `aigne web update` 命令启动。该过程是交互式的，允许您以自然语言提供反馈，然后 WebSmith 会将其转换为对所选页面的结构化更改。

```bash 更新页面 icon=lucide:terminal
aigne web update
```

该命令也可以通过其别名 `up` 调用。

标准工作流程如下：

1.  **执行命令**：运行 `aigne web update`。
2.  **选择页面**：WebSmith 会显示现有页面列表。选择您希望修改的单个页面。
3.  **提供反馈**：输入自然语言指令，描述您想要进行的更改。例如，“将首屏标题更改为‘即时 AI 网站’”或“在‘优势’部分后添加一个三列的功能区。”
4.  **处理**：WebSmith 分析您的请求，并使用其更新工具将更改应用到页面的内容结构中。
5.  **审查并保存**：更新后的页面内容将被保存。您可以重新生成或发布以查看渲染后页面中反映的新文案。

## 内容修改能力

更新工作流使用基于工具的操作。将您的反馈与这些能力对齐，以获得可预测的结果。

| 操作 | 描述 | 反馈示例 |
| :--- | :--- | :--- |
| **添加部分** | 添加一个新的内容区块。指定内容及其位置。 | “在‘功能’部分后添加一个‘客户’部分，包含三条推荐语。” |
| **删除部分** | 按名称删除一个现有区块。 | “移除‘旧版支持’部分。” |
| **更新部分** | 修改区块内的内容。 | “在‘定价’部分，将标题更改为‘为团队提供可预测的定价’。” |
| **移动部分** | 重新排序各部分。 | “移动‘常见问题’，使其出现在最后的 CTA 之前。” |
| **更新元数据** | 更新页面元数据，如标题或描述。 | “将页面标题设置为‘平台概览’。” |

## 反馈提示示例

```text 反馈示例 icon=lucide:clipboard-list
"在首屏之后添加一个四步演练部分，用于入门引导说明。"

"移除引用旧版 API 客户端的案例研究部分。"

"将首屏中的 CTA 更改为‘查看实际效果’，并调整副标题以提及 5 分钟设置。"
```

## 总结

更新页面内容是一个由清晰指令驱动的迭代过程。通过将您的反馈与 WebSmith 的页面更新能力对齐，您可以快速优化文案、重新排序部分，并保持网站的准确性，而不会影响网站的其余部分。

有关结构性更改，请参阅[更新结构](./guides-update-website-update-structure.md)指南。有关发布更新，请访问[发布网站](./guides-publish-website.md)。
