# 更新結構

當您想使用 `aigne web generate` 重塑網站地圖時，例如在批准計畫前新增或移除頁面，請遵循本指南。

修改網站結構——例如新增頁面、移除過時頁面或重組導覽——是保持內容相關性和使用者友好性的關鍵任務。AIGNE WebSmith 透過一個由 AI 驅動的工作流程來簡化此過程，該流程能解讀自然語言回饋以執行精確的結構性變更。

本文件說明如何使用 `aigne web generate` 指令來修改您的網站結構。在此過程中，一個 AI Agent 會解讀您的自然語言回饋，以新增、移除、更新或移動頁面。理解可用的操作有助於您提供清晰的指令，以實現您期望的網站佈局。

應用結構性變更的主要指令是 `aigne web generate`。有關建立過程的更多詳細資訊，請參閱[建立網站](./guides-create-website.md)。
## 結構更新流程

當您執行 `aigne web generate` 指令時，系統會首先分析您的內容來源並提出一個網站結構。接著，您將有機會以互動方式審查和修改此結構。

## 互動式更新工作流程

更新網站結構的過程是互動式的。以下是該工作流程的摘要：

1.  **啟動生成**：在您的終端機中執行生成指令。
    ```bash 執行指令 icon=lucide:terminal
    aigne web generate
    ```

2.  **審查建議的結構**：在分析您的來源後，終端機會顯示建議的結構並請求您的輸入。

3.  **提供自然語言回饋**：在提示符處，以自然語言輸入您想要的變更。例如：
    > "在頂層新增一個 '聯絡我們' 頁面，路徑為 `/contact`。同時，移除路徑為 `/about/old-team` 的頁面。"

4.  **審查並批准**：AI 會處理您的回饋，應用必要的變更，並顯示修改後的結構。您可以批准變更以完成此過程，或提供更多回饋以進一步完善。

## 核心結構性操作

AI Agent 能理解四種修改網站結構的基本操作。提供與這些操作一致的清晰回饋將產生最準確的結果。

| 操作 | 說明 | 關鍵參數 |
| :--- | :--- | :--- |
| **新增 (Add)** | 在網站結構中建立一個新頁面。 | `title`、`description`、`path`、`parentId`、`sourceIds` |
| **刪除 (Delete)** | 從結構中移除一個現有頁面。 | `path` |
| **更新 (Update)** | 修改現有頁面的中繼資料。 | `path`、`title` (選用)、`description` (選用)、`sourceIds` (選用) |
| **移動 (Move)** | 將頁面重新定位到不同的父層級下，並變更其路徑。 | `path`、`newParentId`、`newPath` |

### 回饋範例

以下是一些關於如何表達您的回饋以執行特定結構性變更的範例：

- **新增頁面：**
 > "在 '/about' 區塊下新增一個標題為 '我們的團隊' 的新頁面。路徑應為 `/about/team`。"

- **刪除頁面：**
 > "請移除位於 `/products/legacy` 的 '舊版產品' 頁面。"

- **更新頁面：**
 > "將 `/services` 頁面的標題更新為 '我們的專業服務'。"

- **移動頁面：**
 > "將目前位於 `/careers` 的 '職涯發展' 頁面移動到 '/about' 頁面下。其新路徑應為 `/about/careers`。"

## 總結

這種互動式過程確保您的網站結構始終井然有序、保持最新，並與您的目標一致。系統會處理技術細節，讓您能專注於內容的邏輯安排。
