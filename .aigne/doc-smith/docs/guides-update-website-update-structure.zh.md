# 更新结构

当您想在批准计划前使用 `aigne web generate` 重塑站点地图（例如添加或删除页面）时，请遵循本指南。

修改网站结构——例如添加新页面、删除过时页面或重新组织导航——是保持内容相关性和用户友好性的关键任务。AIGNE WebSmith 通过一个由 AI 驱动的工作流来简化此过程，该工作流能解释自然语言反馈以执行精确的结构性更改。

本文档解释了如何使用 `aigne web generate` 命令修改您的站点结构。在此过程中，AI agent 会解释您的自然语言反馈，以添加、删除、更新或移动页面。了解可用的操作有助于您提供清晰的指令，以实现您期望的站点布局。

应用结构性更改的主要命令是 `aigne web generate`。有关生成过程本身的更多详细信息，请参阅[生成网站](./guides-generate-website.md)。
## 结构更新过程

当您运行 `aigne web generate` 命令时，系统首先会分析您的内容源并提出一个网站结构。然后，您将有机会以交互方式审查和修改此结构。

## 交互式更新工作流

更新网站结构的过程是交互式的。以下是该工作流的摘要：

1.  **启动生成**：在您的终端中运行生成命令。
    ```bash 运行命令 icon=lucide:terminal
    aigne web generate
    ```

2.  **审查建议的结构**：在分析您的源文件后，终端会显示建议的结构并请求您的输入。

3.  **提供自然语言反馈**：在提示符处，用自然语言输入您期望的更改。例如：
    > “在顶层添加一个路径为 `/contact` 的‘联系我们’页面。同时，删除路径为 `/about/old-team` 的页面。”

4.  **审查并批准**：AI 会处理您的反馈，应用必要的更改，并显示修改后的结构。然后，您可以批准更改以完成该过程，或提供更多反馈以进一步完善。

## 核心结构操作

AI agent 理解四种用于修改网站结构的基本操作。提供与这些操作相符的清晰反馈将产生最准确的结果。

| 操作 | 描述 | 关键参数 |
| :--- | :--- | :--- |
| **添加 (Add)** | 在网站结构中创建一个新页面。 | `title`、`description`、`path`、`parentId`、`sourceIds` |
| **删除 (Delete)** | 从结构中移除一个现有页面。 | `path` |
| **更新 (Update)** | 修改一个现有页面的元数据。 | `path`、`title` (可选)、`description` (可选)、`sourceIds` (可选) |
| **移动 (Move)** | 将一个页面重新定位到不同的父级下，从而改变其路径。 | `path`、`newParentId`、`newPath` |

### 反馈示例

以下是一些如何措辞您的反馈以执行特定结构性更改的示例：

- **添加页面：**
 > “在 ‘/about’ 部分下添加一个标题为‘我们的团队’的新页面。路径应为 `/about/team`。”

- **删除页面：**
 > “请移除位于 `/products/legacy` 的‘旧版产品’页面。”

- **更新页面：**
 > “将 `/services` 页面的标题更新为‘我们的专业服务’。”

- **移动页面：**
 > “将当前位于 `/careers` 的‘职业发展’页面移动到‘/about’页面下，使其成为其子页面。新路径应为 `/about/careers`。”

## 总结

这种交互式过程确保您的网站结构始终井然有序、保持最新，并与您的目标保持一致。系统会处理技术细节，让您能专注于内容的逻辑布局。
