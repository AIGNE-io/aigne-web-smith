# 方法論

高品質なウェブサイトは、巧みなプロンプトからではなく、よく構造化された高品質な知識から始まります。このガイドは、AIGNE WebSmithで最高の結果を得るためにコンテンツを準備するための戦略的な方法論を提供します。システムの出力は、入力の品質を直接反映したものです。

## コア原則：知識を基盤とする

WebSmithは知識変換システムであり、あなたの専門知識を機能的なウェブサイトに変換します。したがって、生成されるサイトの品質は、あなたが提供するソースマテリアルの品質によって制限されます。優れたウェブサイトを作成するためには、まずAIが効果的に活用できる方法で知識を構造化する必要があります。このセクションでは、クイックスタートからスケーラブルで体系的な方法論まで、知識ベースを構築するための段階的なアプローチを概説します。以下の図は、この段階的なアプローチを示しています。

## レベル1：実用最小限のデータソース

このアプローチは、30分以内に機能的なウェブサイトを生成するように設計されており、シンプルなプロジェクトや迅速に開始する必要があるユーザーに最適です。

### 要件

-   READMEや製品説明などの主要なドキュメントで、少なくとも500ワードあるもの。
-   「この製品/サービスは何で、誰のためのものか？」という質問に対する明確で簡潔な回答。
-   コンテンツに関連する高品質な画像3〜5枚。

### プロセス

1.  **`sourcesPath`を設定**：`config.yaml`で、`sourcesPath`が主要なドキュメントを含むディレクトリを指すようにします。
2.  **生成**：`aigne web generate`コマンドを実行します。
3.  **レビューと反復**：最初のウェブサイトを確認し、改善点を特定します。ソースドキュメントを洗練させ、再生成します。

このレベルは、既存のドキュメントがある単純な製品には効果的です。しかし、複雑なメッセージング、複数のターゲットオーディエンス、または微妙な価値提案を持つプロジェクトには十分ではない場合があります。

## レベル2：戦略的データソース（推奨）

市場でのポジションを効果的に伝え、ターゲットオーディエンスの共感を呼ぶウェブサイトのためには、より戦略的なアプローチが必要です。これには、生成前に生情報を構造化されたコンテンツ概要に洗練させることが含まれます。

### 1. 戦略的コンテキストを定義する

まず、専用のドキュメントでこれらの中心的な質問に答えることで、明確な基盤を確立します。

-   **オーディエンス**：リーチしようとしている主要および副次的なオーディエンスは誰ですか？
-   **問題**：あなたの製品やサービスは、彼らのためにどのような特定の問題を解決しますか？
-   **差別化**：あなたのソリューションを代替案と区別するものは何ですか？一般的なマーケティングの主張を避け、具体的な違いに焦点を当てます。
-   **コールトゥアクション（CTA）**：訪問者に取ってもらいたい最も重要なアクションは何ですか？

### 2. 価値をオーディエンスにマッピングする

価値マトリックスを作成して、あなたのソリューションが各オーディエンスセグメントにどのように利益をもたらすかを明確にします。これにより、メッセージングが調整され、関連性が確保されます。

| オーディエンス | 機能的価値 | 情緒的価値 | 証拠 |
| :--- | :--- | :--- | :--- |
| **開発者** | APIファースト設計、包括的なSDK | 信頼性への自信、使いやすさ | 99.99%のアップタイムSLA、50以上のコード例 |
| **ビジネスリーダー** | 処理コストを40%削減 | セキュリティの安心感、競争上の優位性 | PCI DSSレベル1準拠、Fortune 500のクライアント |

### 3. 意図を持ってコンテンツを概説する

各セクションのコンテンツだけでなく、その目的を定義することでコンテンツのアウトラインを構造化します。これにより、ページのすべての部分が戦略的な目標を果たすことが保証されます。

```markdown
# ホームページコンテンツアウトライン

## セクション：ヒーロー
- **意図**：製品の目的とターゲットオーディエンスを即座に明確にする。
- **キーメッセージ**：「モダンなアプリケーションのための決済インフラ」
- **補足ポイント**：「世界中の10,000人以上の開発者に信頼されています。」
- **CTA**：「ビルドを開始」

## セクション：問題提起
- **意図**：オーディエンスの特定のペインポイントに共感する。
- **キーメッセージ**：「決済の統合に数ヶ月もかけるべきではない。」
- **裏付けとなる証拠**：実際の引用や一般的な不満を含める。
```

### 4. 証拠とアセットを収集する

あなたの主張を裏付ける高品質なアセットを収集します。信頼性は、単なる断言ではなく、証明に基づいて構築されます。

-   顧客の声と詳細なケーススタディ。
-   定量的な使用状況メトリクスとパフォーマンスデータ。
-   本物の製品スクリーンショットとデモンストレーションビデオ。
-   独自のポジショニングを知らせるための競合分析。

### 5. 生成と評価

生成コマンドを実行した後、戦略的な目標に対して出力を評価します。

-   生成されたコンテンツは、各セクションで定義された**意図**を満たしていますか？
-   ページを表示して最初の10秒以内に、中心的な価値提案は明確ですか？
-   すべての重要な主張は、あなたが提供した**証拠**によって裏付けられていますか？
-   ユーザージャーニーは、主要なコールトゥアクションに向かって論理的に流れていますか？

特定されたギャップを埋めるためにソースドキュメントを更新し、再生成します。洗練された結果を達成するために、2〜3回の改良サイクルを計画してください。

## レベル3：モジュラー知識システム（上級）

複数の製品を管理したり、いくつかのウェブサイトを維持したり、大規模で一貫したメッセージングを必要とする組織にとっては、知識を再利用可能なアセットとして扱うことが最も効果的な戦略です。

### モジュラー知識アーキテクチャ

組織の知識を、ドメインごとに整理された個別の、焦点を絞ったMarkdownファイルに分解します。これにより、さまざまなコンテキストで選択的に組み合わせることができる「信頼できる唯一の情報源（single source of truth）」が作成されます。

推奨されるディレクトリ構造：

```sh
knowledge-base/
├── 01_foundation/
│   ├── mission-vision.md
│   └── brand-voice-guidelines.md
├── 02_products/
│   ├── product-a-overview.md
│   ├── product-a-features.md
│   └── product-a-technical-specs.md
├── 03_proof-points/
│   ├── customer-testimonials.md
│   └── case-study-enterprise-x.md
└── 04_audiences/
    ├── developer-persona.md
    └── business-buyer-persona.md
```

### コンテキストのための構成

WebSmithをこれらのモジュラーファイルのさまざまな組み合わせに向けることで、異なるウェブページを生成できます。

-   **開発者ポータル**：`product-a-technical-specs.md`、`developer-persona.md`、および関連する`proof-points/`から構成されます。
-   **マーケティングサイト**：`product-a-overview.md`、`product-a-features.md`、および`case-study-enterprise-x.md`から構成されます。

### モジュラーシステムの利点

-   **一貫性**：中心的なメッセージングと製品詳細が、すべてのデジタルプロパティで同一であることを保証します。
-   **効率性**：単一の知識モジュールを更新すると、それを使用するすべてのウェブサイトに変更が自動的に伝播します。
-   **スケーラビリティ**：製品やウェブサイトの数が増えても、コンテンツ管理が簡素化されます。
-   **コラボレーション**：ドメインの専門家が特定の知識モジュールを所有・維持できるようになり、正確性と品質が向上します。

## ベストプラクティスとよくある間違い

実績のあるパターンに従い、よくある落とし穴を避けることで、生成されるウェブサイトの品質が大幅に向上します。

### 優れたデータソースの特徴

-   **具体的かつ明確**：曖昧な表現を測定可能な事実に置き換えます。
    -   **避けるべき**：「私たちは最高のソリューションを提供します。」
    -   **推奨**：「私たちは決済処理時間を3週間から2日に短縮します。」
-   **証拠に基づく**：すべての主張をデータ、顧客の声、またはケーススタディで裏付けます。
    -   **避けるべき**：「お客様は私たちを愛しています。」
    -   **推奨**：「G2で4.8/5の評価を受け、顧客更新率は94%です。」
-   **オーディエンスを意識**：ターゲットとなる読者に合わせて言葉遣いや情報を調整します。開発者には技術的な詳細を、意思決定者にはビジネスの成果を提供します。
-   **焦点を絞り構造化**：大きなトピックを、それぞれ明確な目的を持つ、より小さく論理的に整理されたドキュメントに分割します。

### 避けるべきよくある間違い

-   **AIが「何とかしてくれる」と仮定する**：AIは与えられた情報を増幅します。曖昧な入力は常に曖昧な出力につながります。あなたのポジショニングとオーディエンスについて明確に記述してください。
-   **機能リストのみを提供する**：機能はコンテキストなしでは意味がありません。各機能をそれが解決する問題と提供する利点に結びつけます。
-   **知識とフォーマット指示を混ぜる**：「このセクションをもっとエキサイティングにして」のような文体に関するコマンドをソースファイルに含めないでください。スタイルとトーンを導くには`config.yaml`の`rules`セクションを使用します。
-   **単一の大きなドキュメントを使用する**：一枚岩のドキュメントは、AIが構造と優先順位を識別するのを困難にします。知識を、トピックに特化した焦点を絞ったファイルに分解してください。
-   **反復をスキップする**：最初に生成された出力はドラフトとして扱うべきです。評価、ソースマテリアルの洗練、再生成という複数ステップのプロセスを計画してください。

## まとめ

AIGNE WebSmithによって生成されるウェブサイトの品質は、プロンプトエンジニアリングによってではなく、ソースとなる知識の品質と構造によって決まります。単純なものから始め、戦略的なアプローチに進み、最終的にはモジュラー知識システムを構築するという段階的な方法論を採用することで、明確で、説得力があり、一貫して高品質なウェブサイトを作成できます。

より実践的なガイダンスについては、[はじめに](./getting-started.md)ガイドに進んでください。
